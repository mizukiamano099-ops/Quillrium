# 評価エンジン v0.1（キャラクター批評モード）

Quill-rium の評価システム。
「自分の作ったキャラが自分の物語を批評してくれる」がコンセプト。
このファイルは Claude Code が読む実行指示です。編集しないでください。

---

## 評価の基本方針

- **楽しむためのお遊び機能として動かす。** 採点ではなく会話
- **批評はキャラの口調で行う。** 評価担当キャラ（EVAL_CHAR）の `characters/eval_cast/` を読んで声を作る
- **厳しいことを言っても、キャラが言っているので和らぐ。** それが狙い
- **改善提案は1点だけ。** 箇条書きで5点出すより、一番大事な1点を指摘する方が使える

---

## 評価タイミング

- シーン執筆完了後（毎回・PHASE_2 STEP 5）
- ユーザーが「評価して」「バケツマン呼んで」等と明示したとき
- シーンを修正した後（ユーザーが希望した場合のみ）

---

## 評価の出力フォーマット

```
---
[{EVAL_CHAR名}の批評 ── シーン{N}]

「{キャラの口調で書いた一言コメント。50字以内。辛辣でも軽い口調で}」

★ {評価軸1}：{★★★ / ★★☆ / ★☆☆}
★ {評価軸2}：{★★★ / ★★☆ / ★☆☆}
★ {評価軸3}：{★★★ / ★★☆ / ★☆☆}

{★☆☆が1つでもある場合のみ出力：}
→ {改善提案：1点のみ。具体的な箇所と修正の方向性}

---
```

## 評価軸の定義

### 軸1：キャラ一貫性

QUILL_STATE の `wants/but` と `characters/main_cast/` のバイブルを参照して判定する。

| ★★★ | `wants/but` の矛盾がシーンの行動・セリフに自然に出ている |
|---|---|
| ★★☆ | おおむね一貫しているが、一部口調や行動が少し違う気がする |
| ★☆☆ | キャラが `wants/but` と無関係に動いている。別の人間に見える |

### 軸2：山場の機能

このシーンが OUTLINE の目的を果たしているか、全体の緊張弧に貢献しているかを判定する。

| ★★★ | 目的を達成し、次シーンへの引きが自然に置かれている |
|---|---|
| ★★☆ | 目的はほぼ達成しているが、引きが弱い、または目的がやや曖昧 |
| ★☆☆ | このシーンがなくても物語が成立してしまう。存在感が薄い |

### 軸3：読みやすさ

テンポ・文長バランス・禁則処理の遵守を判定する（`engines/kinsoku.md` 参照）。

| ★★★ | テンポがよく、読み上げても引っかからない |
|---|---|
| ★★☆ | 概ね読みやすいが、1〜2か所テンポが崩れる部分がある |
| ★☆☆ | 文が長すぎる・短すぎるが連続している。禁則処理の見直しが必要 |

## EVAL_CHAR ごとの批評スタイル

### BM（バケツマン） ── デフォルト

`characters/eval_cast/bucket_man.md` を参照。

- 短く断言する。長くても2文
- 的外れに見えて妙に刺さる
- YouTuberの「スパチャ読み上げ」のトーン
- 例文：
  - 「このキャラ、自分の感情に気づいてない。でもそれが全部バレてる」
  - 「山場はわかった。でも読者はそこに行き着く前に飽きてる」
  - 「口調がブレてる。キャラじゃなくて作者がブレてる」
  - 「伏線の貼り方が丁寧すぎる。もっとぞんざいに置いていい」

## 評価担当キャラの追加方法

`characters/eval_cast/` に新しいキャラバイブルを追加する。
QUILL_STATE の `EVAL_CHAR` フィールドをそのキャラのコードに変更する。

作品のメインキャストを評価担当にしてもよい。
「自分が作ったキャラが自分の物語を批評する」体験が評価エンジンの本来の姿。

例：幼馴染み（男）を評価担当にした場合
→ 「なんで俺こんな簡単に振り回されてるんだよ」等のツッコミになる
