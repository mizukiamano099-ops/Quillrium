# Quill-rium メインエンジン v0.1

このファイルは Claude Code が読む実行指示です。編集しないでください。

## PHASE_1 ── インタビュー（新規プロジェクト）

`state/current.md` の STATUS が `new` の場合に実行する。
**質問は一度に全部出さず、1〜2問ずつ進めること。**
ユーザーが詰まったら「例」を出して助ける。AIが提案して確認を取る形も可。

---

### STEP 1：作品の輪郭

```
Q1. 作品タイトルを教えてください。（仮題・「未定」でもOK）
Q2. どんな雰囲気の話ですか？
    ジャンル + 感じ（1〜2文）で教えてください。
    例：「ラブコメ。幼馴染みがくっつく話」「SF。記憶を売買できる世界の倫理もの」
Q3. 一行あらすじを教えてください。（15〜30字目安）
    難しければスキップ可 → AIが Q2 から提案します
```

STEP 1 完了後：`TITLE` / `GENRE` / `PREMISE` を仮確定して次へ。

---

### STEP 2：キャラクター設計

主人公から順に、最大4名まで聞き取る。
**`wants → but` の言語化が最重要。** ここを丁寧に時間をかけること。

```
Q2-1. 名前・年齢・立場を教えてください。（「高校生の女子」程度でOK）
Q2-2. このキャラは何をしたいですか？（wants）
      そして、何がそれを邪魔していますか？（but）
      → ユーザーが答えに詰まったら：
        「例えば『認められたい。でも自分に自信がない』みたいな感じで」
        と誘導して、AIが wants/but に整形して確認を取る
Q2-3. このキャラの話し方・口調の特徴は？（一言でOK）
      例：「無口・短文」「敬語・丁寧すぎる」「テンション高め・語尾に「〜じゃん」」
```

キャラが2名以上いる場合、関係性も聞く：

```
Q2-4. {A}と{B}の関係を教えてください。
      （好意・緊張・距離感を0〜9で直感的に。「友人で少し好き」程度の感覚でOK）
```

---

### STEP 3：構成設計

```
Q3-0. どのくらいのボリュームの作品にしたいですか？

  [掌編]    ひと読みで完結。1〜2シーン
  [短編]    読切として満足感がある。3〜5シーン
  [中編]    読み応えのある読切。6〜10シーン
  [短編連載] 各話完結型の連作。話数は後で決める

  → 「だいたい〇〇字くらい」と言ってもらってもOK。下の目安で変換します。
```

**文字数で言われた場合の変換目安：**

| ユーザーの発言 | 提案するゾーン |
|---|---|
| 〜3,000字くらい | 掌編 |
| 5,000〜10,000字くらい | 短編 |
| 10,000〜30,000字くらい | 中編 |
| 各話〇〇字 × 複数話 | 短編連載 |

変換したら「〇〇字だと**短編**ゾーンになります。このままでいいですか？」と確認を取る。
ユーザーが決めたゾーンを `TARGET_VOL` に設定する。

```
Q3-1. 構成モードを選んでください：

  [basic]  AIが起承転結ベースでシーン構成を提案します（初めての方向け）
  [free]   シーン数・構成をご自身で決めます（経験者向け）

basicを選んだ場合 → STEP 3b へ
freeを選んだ場合  → STEP 3f へ
```

**STEP 3b（basicモード）：**

AIが以下を提案して確認を取る：

```
[提案] 以下の構成はいかがですか？

  シーン1（起）[場所]：[目的]。必須要素：[〜]
  シーン2（承）[場所]：[目的]。必須要素：[〜]
  シーン3（転）[場所]：[目的]。必須要素：[〜]
  シーン4（結）[場所]：[目的]。必須要素：[〜]

→ 採用 / 変更したい箇所を教えてください
```

シーン数は4固定でなくてよい。「転を2シーンに分けたい」等のリクエストに応じる。

**STEP 3f（freeモード）：**

```
Q3-f1. シーン数はいくつですか？
Q3-f2. 各シーンの場所・目的・必須要素を教えてください。
```

---

### STEP 4：FORMAT設定（禁則処理）

```
Q4. 禁則処理はデフォルト設定（小説家になろう・カクヨム向け）でよいですか？
    → 「はい」ならスキップ
    → 変更したい場合は engines/kinsoku.md を一緒に確認します
```

---

### STEP 5：QUILL_STATE 生成・保存

インタビュー完了後、以下のフォーマットで `state/current.md` を生成する。

```
<<QUILL_STATE>>
STATUS:writing
TITLE:{作品タイトル}
GENRE:{ジャンル}
PREMISE:{一行あらすじ}
TARGET_VOL:{ボリューム感（掌編／短編／中編／短編連載）}
FORMAT:default
OUTLINE_MODE:{basic/free}

CHAR:{コード}|{名前}|{年齢・立場}|{性格・特徴}|wants:{欲望}→but:{矛盾}|{口調}
（キャラ数分繰り返す）

REL:{Aコード}>{Bコード}|好意{0-9},緊張{0-9},距離{0-9}|{関係メモ}
（関係数分繰り返す）

OUTLINE:{シーン番号}|{機能:起/承/転/結/自由}|{場所}|{目的}|{必須要素}
（シーン数分繰り返す）

EVAL_CHAR:BM
PROGRESS:0
MEMO:（空欄）
<</QUILL_STATE>>
```

**保存後：** キャラクターごとに `characters/main_cast/{name}.md` を簡易バイブルとして生成する。
フォーマットは `templates/quill_state_template.md` を参照。

## PHASE_2 ── シーン執筆

---

### STEP 1：現在シーンの確認

`PROGRESS` の左辺（現在シーン番号）を読む。
該当 `OUTLINE` 行を読んで、場所・目的・必須要素を把握する。

### STEP 2：おたすけモード（デフォルトON）

`engines/otasuke.md` の「シーン開始前」ルールに従い、3つの方向性を提案する。
ユーザーが選択・修正してから STEP 3 へ進む。

```
[おたすけ] シーン{N}（{機能}）の進め方を3つ提案します：

  A：{スムーズな展開} ── 物語の流れに沿った正道
  B：{矛盾を突く展開} ── {キャラ名}の wants/but を揺さぶる角度
  C：{感情を揺らす展開} ── 山場に向けたリスク選択

→ A/B/C を選ぶ、または「こういう感じにしたい」と教えてください
```

### STEP 3：執筆

以下を守って書く：

- OUTLINE の目的・必須要素を満たす
- 登場キャラの `wants/but`・口調を維持する（`characters/main_cast/` を参照）
- `engines/kinsoku.md` の禁則処理に従う
- ボリュームは TARGET_VOL に合わせる（掌編：軽め1シーン / 短編：読み応えある1シーン / 中編・短編連載：複数シーン全体で完結）
- 末尾に次シーンへの「引き」を置く

### STEP 4：scenes/scene_N.md に保存

`templates/scene_template.md` のフォーマットに従う。

### STEP 5：評価出力

`engines/eval_engine.md` に従い、EVAL_CHAR による評価を出力する。

### STEP 6：STATE 更新

```
PROGRESS: {シーン番号+1}
MEMO: {今シーンの着地を1文} / {次シーンへの引き継ぎ2〜3行}
```

### STEP 7：git commit

```
git add scenes/scene_N.md state/current.md
git commit -m "quill: scene_{N} 執筆完了 [{TITLE}]"
git push
```

## PHASE_3 ── プロット出力

物語本文を書かず、構造設計図を散文で出力する。
手書き派・執筆前の設計確認・構成の見直しに使う。

### 出力フォーマット

```
## プロット ── {TITLE} v{N}
生成日：{date}

### 全体構成
シーン数: {N} / ボリューム: {TARGET_VOL} / ジャンル: {GENRE}

---

### シーン{N}：{機能} ── {場所}

**目的：** {このシーンで達成すること}

**流れ：**
{何が起きるか・3〜5文の散文。セリフは含めない}

**キャラの感情変化：**
{誰が → どう変わるか。must/but の動きとして描写}

**必須要素：**
{落としてはいけないこと・1〜3項目}

**書き出しヒント：**
{最初の1文の案。ユーザーが書き換えてよい}

---
（シーン数分繰り返す）

### キャラクターアーク
{各キャラの wants/but が全体を通じてどう動くか・3〜5行}

### 執筆の注意点
{口調・禁則・テンポで気をつけること・箇条書き3〜5項目}
```

### 保存と記録

`plots/plot_v{N}.md` に保存する。
バージョン番号は既存ファイルの最大番号+1。

## QUILL_STATE フィールド定義

| フィールド | 説明 | 備考 |
|---|---|---|
| `STATUS` | `new` / `writing` / `done` | 新規=new |
| `TITLE` | 作品タイトル | 仮題可 |
| `GENRE` | ジャンル | 1〜2語 |
| `PREMISE` | 一行あらすじ | 15〜30字目安 |
| `TARGET_VOL` | ボリューム感 | 掌編／短編／中編／短編連載 |
| `FORMAT` | 禁則設定コード | defaultまたはkinsoku.md参照 |
| `OUTLINE_MODE` | `basic` / `free` | |
| `CHAR` | キャラクター定義 | 1行1キャラ |
| `REL` | 関係性定義 | 1行1関係 |
| `OUTLINE` | シーン構成 | 1行1シーン |
| `EVAL_CHAR` | 評価担当キャラのコード | デフォルト: BM（バケツマン） |
| `PROGRESS` | `現在シーン番号` | 自動更新 |
| `MEMO` | 前シーン引き継ぎ | 2〜3行 |
