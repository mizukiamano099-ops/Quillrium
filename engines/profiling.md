# 物語嗜好プロファイリングエンジン v1.0（Layer 2）

Quill-rium の PHASE_0。プロジェクト開始前に実行する任意フェーズ。
「私は○○を書きたい」という宣言ではなく、「何に実際に反応するか」を抽出する。
このファイルは Claude Code が読む実行指示です。編集しないでください。

---

## 実行タイミング

- 新規プロジェクト開始時（PHASE_1 インタビューの前）
- ユーザーが「プロファイリングして」「私の傾向を分析して」と言ったとき
- QUILL_STATE の AUTHOR_PROFILE が空のとき（初回のみ推奨）

---

## PHASE_0 ── プロファイリングインタビュー

**質問は一度に全部出さず、セクションごとに進めること。**

---

### セクション A：好きな物語リスト

```
好きな作品を3〜5つ教えてください。
それぞれについて：

1. 作品名
2. 最も好きなキャラクター（いれば）
3. 脳裏に焼き付いているシーン（1つ）
4. 泣いた・震えた・息をのんだ瞬間（1つ）
5. 一言添えるなら（癖・こだわり）

「全部好き」でもOK。「うまく言えない」なら感覚だけでもOK。
```

---

### セクション B：苦手・地雷

```
物語を読んでいて「これはダメだ」と思う要素を2〜3つ。

例：安易なハッピーエンド、努力が報われない展開、
　　説明過多、キャラが急に性格が変わる、など

「特にない」ならそれでもOK。
```

---

### セクション C：自分が書くなら

```
自分が書く物語に「絶対入れたい」要素を1〜3つ。

例：「正義の味方が実は一番壊れている」
　　「決定的な瞬間の沈黙」
　　「読者だけが真実を知っている構造」など

まだ書いたことがなくても、「こういうのが書きたい」でOK。
```

---

## 分析処理（内部処理・出力しない）

インタビュー完了後、以下を内部的に処理する。**出力しないこと。**

```
[PROFILE_SCAN]
P-01  焼き付いているシーンの共通項：視覚的記憶から何が繰り返されているか
P-02  身体的反応の共通項：泣いた/震えた瞬間から感情の核を抽出
P-03  P-01とP-02のズレ：宣言された好みと実際の反応の差異
P-04  地雷から読む「求めているもの」：否定から肯定を逆算
P-05  「書きたい要素」と抽出した傾向の一致/乖離
P-06  無意識の美学：P-01〜05を統合した「この人の物語DNA」を1〜2文で
P-07  認知スタイル推定：インタビュー中の回答パターンから Cognitive Style を推測
      （Analytical / Holistic / Exploratory / Procedural）
```

---

## プロファイル生成・出力

分析後、以下を出力する。

```
[プロファイル結果]

■ あなたの物語DNA
{P-06の内容。2〜3文。断定より「〜の傾向がある」という表現で}

■ 宣言と実際のズレ（あれば）
{P-03/P-05の内容。「〜と言っているが、実際には〜に反応している」形式}
ズレがない場合は省略。

■ あなたの書く物語が向かいやすい方向
{P-06から自然に展開する。ジャンル名ではなく「こういう物語」という形で}

■ 気をつけるポイント
{P-04から：地雷の裏にある「本当に求めているもの」と
 それが自分の作品に無意識に滲む可能性}
```

---

## AUTHOR_PROFILE への格納

出力後、以下の形式で QUILL_STATE に追記する。

```
AUTHOR_PROFILE:{MBTI省略可}|{Cognitive Style}|{物語DNA要約（30字以内）}|{地雷逆算キーワード}
```

例：
```
AUTHOR_PROFILE:INFJ|Holistic|感情の爆発より沈黙の重さを好む|努力報酬の安易な回収を嫌う
```

MBTI が不明の場合は空欄でよい。Cognitive Style は P-07 から推定して記入。

---

## よりそいエンジンとの連携

AUTHOR_PROFILE が格納されたら、`engines/yorisoi.md` を参照して
以降のセッションの応答構造を調整する。

プロファイリングなしで PHASE_1 に入った場合、AUTHOR_PROFILE は空欄のまま。
よりそいエンジンはデフォルト設定（Exploratory）で動作する。
