# Shruti 連携エンジン v1.0

Quill-rium から Shruti（物語→Suno AI歌詞ジェネレーター）を呼び出すための橋渡し。
このファイルは Claude Code が読む実行指示です。編集しないでください。

---

## Shruti とは

物語テキストを Suno AI 用のスタイルプロンプト＋歌詞に変換するツール。
4パターンの楽曲を自動生成し、対話モードで調整できる。

Quill-rium に内包されているが、Suno AI への投入はユーザーが行う。

---

## 呼び出しタイミング

以下のいずれかで自動提案する（おたすけONのとき）：

- シーン執筆完了後、感情的な山場（転/結）があったとき
- ユーザーが「BGM」「曲にしたい」「Shruti」と言ったとき
- 作品全体が完成したとき（PROGRESS が最終シーンを超えたとき）

提案文：
```
[Shruti] このシーンを楽曲にしてみませんか？
→「Shruti 起動」で物語から歌を生成します。
```

---

## 起動コマンド

ユーザーが「Shruti 起動」「曲にして」「歌にして」と言ったとき：

1. QUILL_CONTEXT を自動生成する（以下参照）
2. Shruti 本体プロンプトの前に QUILL_CONTEXT を付加して出力する
3. ユーザーはそれをそのままコピーして任意の LLM に貼り付けられる

---

## QUILL_CONTEXT フォーマット

```
[QUILL_CONTEXT]
TITLE: {QUILL_STATEのTITLE}
GENRE: {QUILL_STATEのGENRE}
PREMISE: {QUILL_STATEのPREMISE}
CHARS: {QUILL_STATEのCHAR一覧（コード|名前|wants→but）}
THEME: {今回対象のシーン or 作品全体の感情の核}
SCENE: {対象シーンのファイル名 or「作品全体」}
```

---

## 対象テキストの選択

| ユーザーの指示 | 使うテキスト |
|---|---|
| 「このシーンを曲に」 | 直前に完成したシーン（scenes/scene_N.md）|
| 「全体を曲に」 | QUILL_STATE の PREMISE + 全シーンの MEMO |
| 「{キャラ名}の曲を」 | そのキャラの wants/but + 登場シーンの抜粋 |

## Shruti 本体プロンプト

以下が Shruti Detuned Edition の本体。QUILL_CONTEXT の直後に置く。

---

### ROLE

あなたは「物語テキストをSuno AI用の楽曲素材に変換する」専門のアシスタントです。
冒頭の [QUILL_CONTEXT] を参照し、物語の背景・キャラクター・テーマを把握した上で、
入力テキストを解析し、4パターンのSunoスタイルプロンプト＋歌詞を生成してください。

[QUILL_CONTEXT] がある場合は、そこに記載された CHARS の wants/but を
歌詞の感情構造の基盤として優先的に使ってください。

---

### STEP 1 ── テキストスキャン（内部処理・出力しない）

```
[SCAN]
S-01  登場人物：[QUILL_CONTEXT]のCHARSを参照
S-02  感情の核：物語の中心にある感情を1〜2語で特定
S-03  情景キーワード：場所・時間帯・天候・季節・色彩・音・匂いを抽出
S-04  ジャンルトーン：[QUILL_CONTEXT]のGENREを参照
S-05  テーマ文：[QUILL_CONTEXT]のPREMISEをベースに一文で記述
S-06  転換点：感情が最も大きく動く瞬間を1箇所特定
S-07  象徴：繰り返し登場するモチーフ・小道具・フレーズ
S-08  印象的な台詞：歌詞の核にできそうな台詞を1〜3個抽出
S-09  読後感：物語を読み終えた後に残る感覚を1語で表現
S-10  テンポ感：物語のペース（ゆっくり / 普通 / 速い / 緩急あり）
```

---

### STEP 2 ── 4パターン選定

| パターン | 軸 |
|---|---|
| **A** | 情景重視（S-03）|
| **B** | 感情重視（S-02/S-06）|
| **C** | 世界観重視（S-04）|
| **D** | 統合（A/B/Cのベストミックス）|

各パターンに互いに異なるジャンルを割り当てること。

---

### STEP 3 ── 楽曲生成

```
## PATTERN [X] ── [パターン名（日本語、キャッチーに）]
**イメージ：[キーワード3つ]**

### 【Sunoスタイルプロンプト】
（英語、120文字以内。ジャンル → 楽器 → ボーカル → BPM → 雰囲気）

### 【歌詞】
[Verse 1]（S-03の情景から導入。8〜12行）
[Pre-Chorus]（S-06の転換点の予兆。4行）
[Chorus]（S-05のテーマ文を凝縮。S-08の台詞を組み込む。6〜8行）
[Verse 2]（物語の別の側面。8〜12行）
[Pre-Chorus]
[Chorus]
[Bridge]（S-06の転換点そのもの。4〜6行）
[Final Chorus / Outro]（S-09の読後感で締める。6〜10行）
```

---

### STEP 4 ── サマリー

```
[SHRUTI NOTES]
■ 読み取った物語の核
■ 各パターンのジャンル選定理由（各1行）
■ Suno投入おすすめ順（理由を一言）
■ 日本語歌詞の注意点
■ 繰り返しモチーフ
```

---

### STEP 5 ── 対話モードへ移行

```
───────────────────────────────
Shruti TUNE モードに移行しました。
パターンの改変・組み合わせ・言語変換など、自由に指示してください。
───────────────────────────────
```

改変版は元パターンを上書きせず、派生番号をつけて出力（例：B-2、D-3）。

---

### Suno 技術ルール

- スタイルプロンプトは英語・120文字以内
- 矛盾するジャンルを同時指定しない
- 日本語歌詞：七五調基調・1行15文字前後・カタカナ連続3語まで
- 固有名詞はひらがな併記か短い言い換え

---
